# 🏥 RehabDevice — Автономная система мониторинга реабилитации кисти (ESP32)

**RehabDevice** — это профессиональный медицинский IoT-комплекс на базе микроконтроллера **ESP32** и 6-осевого инерциального датчика **MPU6050 / MPU6500 (GY-521)**. Система предназначена для отслеживания подвижности суставов кисти в реальном времени, автоматической детекции тремора и сбора расширенной аналитики.

Устройство работает полностью автономно без подключения к интернету: создаёт собственную Wi-Fi точку доступа (`SoftAP`) с поддержкой **Captive Portal** (автоматическое всплывающее окно авторизации при подключении) и хранит всю историю тренировок во встроенной флэш-памяти чипа (`LittleFS`).

---

## 🛠 1. Аппаратное подключение (Распиновка)

Для сборки проекта требуется плата **ESP32 DevKit** (или аналогичная на чипе ESP-WROOM-32) и модуль гироскопа/акселерометра **GY-521 (MPU-6050 / MPU-6500)**.

| Вывод датчика (GY-521) | Пин ESP32 | Описание назначения |
| :--- | :--- | :--- |
| **VCC** | **3V3** (3.3V) | Питание датчика (рекомендуется стабильные 3.3V от платы ESP32) |
| **GND** | **GND** | Общая земля |
| **SCL** | **GPIO 22** | Линия тактирования шины I2C |
| **SDA** | **GPIO 21** | Линия данных шины I2C |
| **INT** | **GPIO 19** | Аппаратное прерывание готовности пакетов процессора DMP |
| **AD0** | **GND** | Притянуть к земле для фиксации I2C-адреса `0x68` |

## 📂 2. Пошаговая установка библиотек и плагинов для Arduino IDE 2.x

### Шаг 1. Установка библиотек через Менеджер библиотек Arduino IDE
Откройте **Arduino IDE 2.x**, нажмите на иконку «Книги» слева в панели (или нажмите **`Ctrl + Shift + I`**), чтобы открыть **Library Manager (Менеджер библиотек)**. Найдите по поиску и установите **точно следующие версии библиотек**:

1. **`ArduinoJson`** (автор: **Benoit Blanchon**, установленная версия: **`7.4.3`** или новейшая)
   * *Назначение:* Быстрая сериализация и десериализация JSON-пакетов для WebSocket и сохранения файлов сессий.
2. **`Async TCP`** (автор: **ESP32Async**, установленная версия: **`3.4.10`** или новейшая)
   * *Назначение:* Асинхронный сетевой стек TCP для ESP32.
3. **`ESP Async WebServer`** (автор: **ESP32Async**, установленная версия: **`3.11.2`** или новейшая)
   * *Назначение:* Асинхронный HTTP-сервер и обработчик двустороннего WebSocket-соединения с частотой обновления ~30 Гц.
4. **`MPU6050`** (автор: **Electronic Cats**, версия **`1.3.1`** или новейшая)
   * *Назначение:* Официальная библиотека от Electronic Cats / Jeff Rowberg с полной поддержкой встроенного сопроцессора **DMP (Digital Motion Processor)** и 6-осевых кватернионов.

---

### Шаг 2. Установка плагина для загрузки файлов веб-интерфейса (`LittleFS Filesystem Uploader`)
Файлы веб-интерфейса (`index.html`, `style.css`, `app.js`) находятся в папке `data` внутри проекта и загружаются прямо во внутреннюю память чипа ESP32 (`LittleFS`). Для этого необходим плагин для Arduino IDE 2.x:

1. Перейдите на официальную страницу плагина: [arduino-littlefs-upload (Earle Philhower)](https://github.com/earlephilhower/arduino-littlefs-upload).
2. Откройте вкладку **[Releases](https://github.com/earlephilhower/arduino-littlefs-upload/releases)** и скачайте файл расширения **`.vsix`** (например, `arduino-littlefs-upload-1.6.3.vsix`).
3. Скопируйте скачанный файл в папку расширений Arduino IDE:
   * **Windows:** `C:\Users\<Ваше_Имя>\.arduinoIDE\plugins\` (если папки `plugins` нет — создайте её и поместите туда файл, либо перенесите в `C:\Users\<Ваше_Имя>\.arduinoIDE\extensions\`).
4. Перезапустите **Arduino IDE 2.x**. Теперь в палитре команд (`Ctrl + Shift + P` или `F1`) появится команда **`Upload LittleFS to Pico/ESP8266/ESP32`**.

---

### Шаг 3. Настройка платы в меню Arduino IDE
Перед прошивкой выберите в верхнем меню **Инструменты (Tools)** следующие параметры:
* **Плата (Board):** `ESP32 Dev Module` (или ваша плата ESP32).
* **Схема разделов (Partition Scheme):** **`Default 4MB with spiffs (1.2MB APP / 1.5MB SPIFFS/LittleFS)`** или **`Huge APP (3MB No OTA / 1MB FS)`**. *(Обязательно должна быть выделена память под файловую систему).*
* **Скорость загрузки (Upload Speed):** `921600` (или `115200` при использовании длинного кабеля).

## 🚀 3. Как прошить и запустить проект

1. **Загрузка прошивки (C++ код):**
   * Откройте скетч `RehabDevice.ino` в Arduino IDE.
   * Нажмите кнопку **Загрузить (`Ctrl + U`)**.
2. **Загрузка веб-интерфейса (папка `/data`):**
   * **ВАЖНО:** Закройте вкладку **Монитор порта (Serial Monitor)** внизу экрана, иначе Windows выдаст ошибку `PermissionError(13, 'Access is denied')`, так как порт будет занят.
   * Нажмите на клавиатуре **`Ctrl + Shift + P`** (или **`F1`**), введите в поиске `LittleFS` и выберите команду **`Upload LittleFS to Pico/ESP8266/ESP32`**. Плагин автоматически соберёт образ из папки `data` и прошит его в память чипа.
3. **Первый запуск:**
   * Откройте Serial Monitor на скорости `115200` бод и перезагрузите ESP32 кнопкой `EN` (Reset) на плате.
   * Вы увидите лог успешной калибровки DMP и старта Wi-Fi.
   * Подключитесь с телефона, планшета или ноутбука к сети **`RehabDevice_AP`** (пароль: **`rehab1234`**).
   * Автоматически всплывёт окно авторизации (Captive Portal), либо перейдите в браузере по адресу **`http://192.168.4.1`**.

## ⚙️ 4. Архитектура и принцип работы системы

Код проекта разбит на 4 независимых высокопроизводительных модуля, работающих без блокировки основного цикла (`loop()`):

```
+-------------------------------------------------------------------------+
|                              ESP32 CORE                                 |
|                                                                         |
|  +-----------------+       +-------------------+       +-------------+  |
|  |   SensorMPU     | ----> |  AnalyticsEngine  | ----> |  MemoryFS   |  |
|  | (DMP 6-Axis,    |       | (Амплитуда, тремор|       | (JSON файлы |  |
|  |  FIFO буфер)    |       |  сгибания, удерж.)|       |  сессий)    |  |
|  +-----------------+       +-------------------+       +-------------+  |
|          |                           |                        |         |
|          v                           v                        v         |
|  +-------------------------------------------------------------------+  |
|  |                       WebServerModule (Async)                     |  |
|  |           (WebSocket трансляция ~30 Гц + Captive Portal)          |  |
|  +-------------------------------------------------------------------+  |
+-------------------------------------------------------------------------+
                                       |
                                       v  (Wi-Fi 192.168.4.1)
                            [Смартфон / Планшет Врача]
```

### 🔹 1. Модуль опроса датчика (`SensorMPU.cpp`)
* Работает с использованием аппаратного сопроцессора **DMP (Digital Motion Processor)** внутри чипа MPU6050/6500 на частоте 100 Гц.
* При готовности новых данных датчик выдаёт прерывание на пин `INT` (`GPIO 19`). ESP32 мгновенно вычитывает FIFO-буфер по шине I2C на скорости **400 кГц** и преобразует кватернионы в точные углы Эйлера (Roll, Pitch, Yaw).
* Предусмотрена **мгновенная рекалибровка 0° на лету** прямо из веб-интерфейса (кнопка принимает текущее положение руки за нулевую точку отсчёта).

### 🔹 2. Аналитический движок (`AnalyticsEngine.cpp`)
В режиме реального времени обрабатывает поток углов и вычисляет медицинские показатели пациента:
* **Амплитуда и границы:** Фиксирует минимальный и максимальный угол наклона кисти за тренировку.
* **Индекс плавности (детекция тремора):** Отслеживает рывки углового ускорения (`d(omega)/dt`). Любое резкое содрогание или тремор мгновенно фиксируется и отражается в проценте плавности движений.
* **Адаптивный детектор сгибаний:** Использует алгоритм гистерезиса для подсчёта реальных повторений упражнения, исключая ложные срабатывания от мелкой дрожи.
* **Время удержания:** Подсчитывает время в секундах, проведённое в крайних точках амплитуды (изометрическая нагрузка).

### 🔹 3. Хранилище сессий (`MemoryFS.cpp`)
* Каждая тренировка автоматически сохраняется как отдельный самодостаточно оформленный JSON-файл в папке `/sessions/` в файловой системе `LittleFS` (имя файла формируется на основе временной метки, а ФИО пациента `patientId` сохраняется внутри JSON).
* **Кольцевой буфер:** Перед каждой новой записью система контролирует свободное место. Если память заполнена более чем на 90%, самый старый файл сессии автоматически удаляется для освобождения места.

### 🔹 4. Веб-сервер и интерфейс (`WebServerModule.cpp` + `/data`)
* Построен на базе асинхронного сервера с поддержкой **WebSocket (`/ws`)** и полной адаптивностью под мобильные устройства и встроенный браузер iOS (`color-scheme: dark`, защита от отскока `rubber-banding`).
* **Вкладка «Тренировка и Мониторинг» (Пациент):** Отображает интерактивный круг с перекрестием, который плавно и без задержек вращается вслед за рукой на скорости ~30 кадров в секунду, а также сетку живых метрик.
* **Вкладка «Интерфейс Врача»:** Предлагает сводный столбчатый график (`Chart.js`) с динамикой амплитуды, плавности и сгибаний по дням, таблицу сохранённых сессий и возможность **мгновенно скачать всю базу данных в формате CSV** для дальнейшего анализа в Excel.
