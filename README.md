# 🏥 RehabDevice — Автономна медична ІоТ-система моніторингів реабілітації кисті (ESP32)

**RehabDevice** — це професійний автономний медико-інженерний IoT-комплекс на базі мікроконтролера **ESP32** та 6-осьового інерційного датчика **MPU6050 / MPU6500 (GY-521)**. Система призначена для точного відстеження рухомості суглобів кисті в реальному часі, автоматичного визначення плавности/тремору та збору розширеної клінічної аналітики.

Весь користувацький веб-інтерфейс (UI), медичні показники, інтерактивні підказки, модальні вікна та експортовані CSV-звіти працюють **виключно українською мовою** 🇺🇦 з урахуванням сучасної медичної термінології та клінічних вимог.
Водночас усі **системні лоґи (`Serial` та `console.log`) та інженерні коментарі у вихідному коді C++/JavaScript/CSS** стандартизовані та виконані **англійською мовою (English)** 🇬🇧 відповідно до найкращих практик інженерної розробки та підтримки вбудованих систем.

Устрій є повністю незалежним від зовнішнього інтернету: він створює власну Wi-Fi точку доступу (`RehabDevice_AP`) з підтримкою **Captive Portal** (автоматичне спливаюче вікно авторизації на iOS та Android при підключенні) і зберігає всю історію тренувань у вбудованій флеш-пам'яті чипа (`LittleFS`) за допомогою високоефективної архітектури потокового доступу.

## 🛠 1. Апаратне підключення (Розпіновка датчика GY-521)

Для складання пристрою використовується плата **ESP32 DevKit** (чип ESP-WROOM-32) та модуль гіроскопа/акселерометра **GY-521 (MPU-6050 / MPU-6500)**.

| Вивід датчика (GY-521) | Пін ESP32 | Опис призначення |
| :--- | :--- | :--- |
| **VCC** | **3V3** (3.3V) | Живлення датчика (рекомендується стабільна напруга 3.3V від плати ESP32) |
| **GND** | **GND** | Спільна земля |
| **SCL** | **GPIO 22** | Лінія тактування апаратної шини I2C |
| **SDA** | **GPIO 21** | Лінія даних апаратної шини I2C |
| **INT** | **GPIO 19** | Апаратне переривання готовності пакетів сопроцесора DMP (Digital Motion Processor) |
| **AD0** | **GND** | Притягнути до землі для фіксації I2C-адреси `0x68` |

## 📂 2. Встановлення бібліотек та налаштування Arduino IDE 2.x

### Крок 1. Обов'язкові бібліотеки (Library Manager)
Відкрийте **Arduino IDE 2.x**, натисніть `Ctrl + Shift + I` (Менеджер бібліотек) та встановіть такі версії:

1. **`ArduinoJson`** (автор: **Benoit Blanchon**, версія **`7.4.3`** або новіша)
   * *Призначення:* Швидка робота з JSON, серіалізація чанків та збереження метаданих пацієнтів.
2. **`Async TCP`** (автор: **ESP32Async**, версія **`3.4.10`** або новіша)
   * *Призначення:* Асинхронний мережевий стек TCP/IP для ESP32.
3. **`ESP Async WebServer`** (автор: **ESP32Async**, версія **`3.11.2`** або новіша)
   * *Призначення:* Асинхронний HTTP-сервер та обробник двостороннього WebSocket-зв'язку (`/ws`) з частотою оновлення ~30 Гц.
4. **`MPU6050`** (автор: **Electronic Cats**, версія **`1.3.1`** або новіша)
   * *Призначення:* Офіційна бібліотека від Electronic Cats / Jeff Rowberg з підтримкою сопроцесора **DMP (Digital Motion Processor)** та кватерніонів.

### Крок 2. Плагін для завантаження файлової системи (`LittleFS Filesystem Uploader`)
Всі файли веб-інтерфейсу (`index.html`, `style.css`, `app.js`) розміщені у теці `/data` і прошиваються прямо у внутрішню пам'ять чипа:
1. Завантажте файл розширення `.vsix` з офіційного репозиторію: [arduino-littlefs-upload (Earle Philhower)](https://github.com/earlephilhower/arduino-littlefs-upload/releases).
2. Скопіюйте `.vsix` у директорію плагінів Arduino IDE (`C:\Users\<Ім'я>\.arduinoIDE\plugins\` або `\extensions\`).
3. Після перезапуску IDE у палітрі команд (`Ctrl + Shift + P` або `F1`) стане доступною команда **`Upload LittleFS to Pico/ESP8266/ESP32`**.

### Крок 3. Налаштування плати в меню «Інструменти» (Tools)
* **Плата (Board):** `ESP32 Dev Module`
* **Схема розділів (Partition Scheme):** **`Default 4MB with spiffs (1.2MB APP / 1.5MB SPIFFS/LittleFS)`** або **`Huge APP (3MB No OTA / 1MB FS)`** *(Обов'язково з виділенням пам'яті під файлову систему)*.
* **Швидкість завантаження (Upload Speed):** `921600`.

## 🚀 3. Як прошити та запустити комплекс

1. **Завантаження C++ прошивки:**
   * Відкрийте скетч `RehabDevice.ino`, натисніть `Ctrl + U` (Завантажити).
2. **Прошивка веб-інтерфейсу:**
   * **ВАЖЛИВО:** Обов'язково закрийте вкладку **Монітор порту (Serial Monitor)**, інакше Windows заблокує COM-порт з помилкою `Access is denied`.
   * Натисніть `Ctrl + Shift + P`, введіть `LittleFS` та оберіть **`Upload LittleFS to Pico/ESP8266/ESP32`**.
3. **Перше підключення:**
   * Увімкніть живлення або натисніть `Reset (EN)` на платі. У Serial Monitor (`115200` бод) з'явиться повідомлення про успішне калібрування DMP та старт Wi-Fi.
   * Підключіться зі смартфона або ноутбука до Wi-Fi мережі **`RehabDevice_AP`** (пароль: **`rehab1234`**).
   * Відкриється автоматичне вікно Captive Portal, або перейдіть за адресою **`http://192.168.4.1`**.

## 🏗 4. Новітня інженерна архітектура системи

Код проєкту побудований за принципом промислових систем реального часу і розділений на 4 незалежні асинхронні модулі:

```
+-----------------------------------------------------------------------------------+
|                                   ESP32 CORE                                      |
|                                                                                   |
|  +------------------+     +--------------------+     +-------------------------+  |
|  |    SensorMPU     | --> |  AnalyticsEngine   | --> |        MemoryFS         |  |
|  |  (DMP 6-Axis,    |     | (Амплітуда, тремор,|     |  (JSONL пацієнт-центричне  |  |
|  |   FIFO 100 Hz)   |     |  згинання, утриман)|     |   сховище + Murmur-хеш) |  |
|  +------------------+     +--------------------+     +-------------------------+  |
|           |                         |                             |               |
|           v                         v                             v               |
|  +-----------------------------------------------------------------------------+  |
|  |                           WebServerModule (Async)                           |  |
|  |        (WebSocket стрімінг чанками ~4500 зап/с + Captive Portal)            |  |
|  +-----------------------------------------------------------------------------+  |
+-----------------------------------------------------------------------------------+
                                      |
                                      v (Wi-Fi 192.168.4.1)
                           [Смартфон / Планшет Лікаря]
```

### 🔹 1. Підсистема сенсорики (`SensorMPU.cpp`)
* Використовує апаратний блок **DMP (Digital Motion Processor)** всередині MPU6050 на частоті 100 Гц, розвантажуючи центральний процесор ESP32 від складних тригонометричних розрахунків.
* Зчитування FIFO-буфера відбувається по I2C на швидкості **400 кГц** по перериванню (`GPIO 19`). Кватерніони перетворюються на точні кути Ейлера (Roll, Pitch, Yaw).
* Підтримується **миттєве рекалібрування 0° на льоту** прямо з веб-інтерфейсу.

### 🔹 2. Клінічний аналітичний рушій (`AnalyticsEngine.cpp`)
В реальному часі аналізує динаміку рухів та розраховує ключові біомеханічні параметри:
* **Амплітуда та межі рухомості (`minAngle` / `maxAngle` / `amplitude`):** Точна фіксація кутових меж за сесію.
* **Індекс плавності рухів (`smoothness`):** Алгоритм диференціювання кутового прискорення (`d(omega)/dt`). Будь-який тремор, спазм або різкий ривок миттєво фіксується та знижує відсоток плавності.
* **Адаптивний детектор згинань (`flexionsCount`):** Використовує подвійний гістерезис для підрахунку дійсних повторень вправ, ігноруючи фізіологічний тремор рук.
* **Час утримання (`holdingTime`):** Фіксація часу перебування пацієнта в крайніх точках амплітуди (ізометричне навантаження).

## ⚡ 5. Інноваційна система зберігання та передачі даних (Zero-Copy Chunked Streaming)

У системі реалізовано передові архітектурні рішення, що повністю усувають ризик переповнення оперативної пам'яті (RAM/Heap) навіть при багаторічній клінічній експлуатації:

### 🗂 1. Пацієнт-центричне сховище JSONL (`/p/<ID>.jsonl` + Murmur-хешування)
Замість тисяч окремих дрібних файлів на флешці, які уповільнюють файлову систему, ми використовуємо **JSONL (JSON Lines)**:
* Для кожного пацієнта створюється один єдиний файл `/p/<ID>.jsonl`. Перший рядок містить метадані пацієнта (`{"name":"..."}`), а всі наступні рядки — це миттєво дописовані запису сесій ($O(1)$).
* В оперативній пам'яті ESP32 зберігається лише мініатюрний **32-бітний Murmur/FNV-1a хеш-індекс (`hashToIds`)**, який зв'язує ПІБ пацієнта з номером файлу. Навіть при сотнях пацієнтів індекс займає менше 2–4 КБ RAM!
* Працює інтелектуальне кільцеве очищення: при заповненні LittleFS > 90% автоматично видаляються найстаріші записи з кінця черги.

### 🌊 2. Потокова видача чанками по WebSocket (Zero-Copy Streaming — Вариант В)
Замість формування монолітного JSON-рядка на 30–40 КБ в пам'яті ESP32 (що викликає фрагментацію та `Out of Memory`), прошивка використовує **потоковий генератор чанків (`streamSessionsToCallback`)**:
* При запиті списку сесій мікроконтролер читає рядки з LittleFS у невеликий буфер на стеку (`512` байт) через `fastReadLine` і кожні **12 сесій (~1.5 КБ)** одразу відправляється порцією по WebSocket (`sessionsStreamChunk`).
* Під час відправки кожного чанка викликається `yield()`, що дає мережевому стеку LwIP час випустити пакети в радіоефір без переповнення TCP-вікна.
* **Рівень витрат RAM ESP32 під час видачі будь-якого обсягу бази становить рівно 1.5 КБ!** Пропускна здатність сягає **~4 500 записів на секунду**.
* Браузер (`app.js`) приймає `sessionsStreamStart`, збирає чанки у буфер пам'яті смартфона та при отриманні `sessionsStreamEnd` виконує хронологичне сортування за 0.1 мс.

## 🖥 6. Можливості веб-інтерфейсу (Лікар та Пацієнт)

* **Повна українська локалізація 🇺🇦:** Всі назви вкладок, сповіщення, модальні вікна та заголовки CSV-файлу оформлені чистою українською мовою.
* **Вкладка «Пацієнт (Тренування)»:**
  * Інтерактивне коло з перехрестям, яке плавно синхронізується з рухом кисті на швидкості 30 кадрів/сек.
  * Панель живих метрик (мін/макс кут, амплітуда, плавність, швидкість, кількість згинань та час утримання).
* **Вкладка «Кабінет Лікаря»:**
  * **Розумний пошук та інтерактивні фільтри (Smart Search):** Швидке фільтрування за точним або частковим ПІБ пацієнта (`.includes()`), а також миттєва вибірка кліком по «піллінгах» (панелі кнопок пацієнтів). При відсутності результатів виводиться адаптивна картка стану з підказками.
  * **Графік без стискання (Zero-Squish Charting):** Стовпчаста діаграма (`Chart.js`) відображає амплітуду, плавність та кількість згинань. Завдяки динамічному контейнеру (`#chartInnerScroll`) кожному стовпцю виділяється **мінімум 58 пікселів ширини**. Якщо сесій багато, графік не плющиться, а стає зручно прокручуваним пальцем (`overflow-x: auto` з підтримкою touch-скролла).
  * **100% Автономна HTML5 Canvas-відмальовка:** Якщо пристрій запущено в ізольованому середовищі Captive Portal без доступу до інтернету (де CDN Chart.js недоступний), система автоматично перемикається на вбудований автономний графічний рушій на чистому Canvas!
  * **Управління даними та експорт:** Можливість миттєвого завантаження повної історії у форматі **CSV (`uFEFF` UTF-8 BOM)** для Excel, а також точкове видалення окремих помилкових записів або цілих пацієнтів.
  * **Сумісність з Apple/Android Captive Portal:** Замість системного `window.confirm` (який заблокований у вбудованих браузерах iOS/Android) використовуються власні стильні модальні вікна підтвердження (`custom-modal-overlay`).
