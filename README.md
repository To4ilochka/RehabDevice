# 🏥 RehabDevice — Автономная система мониторинга реабилитации кисти (ESP32)

**RehabDevice** — это медицинский IoT-комплекс на базе микроконтроллера **ESP32** и 6-осевого инерциального датчика **MPU6050 / MPU6500 (GY-521)**. Система предназначена для отслеживания динамики восстановления подвижности суставов кисти в реальном времени, автоматической детекции тремора и сбора аналитической статистики.

Устройство работает полностью автономно: создаёт собственную Wi-Fi сеть (`SoftAP`) с поддержкой **Captive Portal** (автоматическое всплывающее окно авторизации при подключении) и хранит всю историю тренировок во внутренней флэш-памяти чипа (`LittleFS`).

---

## 🛠 1. Аппаратное подключение (Распиновка)

Для работы требуется плата **ESP32 DevKit** (или аналогичная на чипе ESP-WROOM-32) и модуль гироскопа/акселерометра **GY-521 (MPU-6050 / MPU-6500)**.

| Вывод датчика (GY-521) | Пин ESP32 | Описание назначения |
| :--- | :--- | :--- |
| **VCC** | **3V3** (3.3V) | Питание датчика (рекомендуется 3.3V от стабилизатора ESP32) |
| **GND** | **GND** | Общая земля |
| **SCL** | **GPIO 22** | Линия тактирования шины I2C |
| **SDA** | **GPIO 21** | Линия данных шины I2C |
| **INT** | **GPIO 19** | Аппаратное прерывание готовности пакетов DMP |
| **AD0** | **GND** | Притянуть к земле для фиксации I2C-адреса `0x68` |

## 📂 2. Пошаговая установка библиотек и плагинов

### Шаг 1. Установка зависимых библиотек в Arduino IDE
Установите или скопируйте следующие библиотеки в папку библиотек (`C:\Users\<Ваше_Имя>\Documents\Arduino\libraries\`) или через менеджер библиотек в Arduino IDE:

1. **I2Cdev & MPU6050 (Electronic Cats / Jeff Rowberg)** — библиотека для работы с датчиком и загрузки прошивки аппаратного сопроцессора **DMP (Digital Motion Processor)**.
2. **ESPAsyncWebServer** (`mathieucarbou` или `me-no-dev`) + **AsyncTCP** — асинхронный веб-сервер и обработчик WebSocket-соединений.
3. **ArduinoJson (v6.x / v7.x)** — сериализация и десериализация JSON-сообщений и файлов сессий.

---

### Шаг 2. Установка плагина для прошивки файлов в LittleFS
Так как веб-интерфейс (`index.html`, `style.css`, `app.js`) хранится в отдельной папке `/data` и записывается прямо во встроенную файловую систему чипа, вам понадобится инструмент загрузки:

* **Для Arduino IDE 2.x:** Используйте плагин [arduino-littlefs-upload](https://github.com/earlephilhower/arduino-littlefs-upload) (устанавливается согласно инструкции репозитория).
* **Для Arduino IDE 1.8.x:** Распакуйте архив инструмента `ESP32FS` в папку `C:\Users\<Ваше_Имя>\Documents\Arduino\tools\ESP32FS\tool\esp32fs.jar`.

---

### Шаг 3. Настройка платы в меню Arduino IDE
Перед прошивкой выберите правильные параметры в меню **Инструменты (Tools)**:
* **Плата (Board):** `ESP32 Dev Module` (или ваша конкретная модель ESP32).
* **Схема разделов (Partition Scheme):** `Default 4MB with spiffs (1.2MB APP / 1.5MB SPIFFS/LittleFS)` или `Huge APP`. *Параметр должен выделять место под файловую систему (не менее 1 МБ).*
* **Скорость порта (Upload Speed):** `921600` (или `115200` при нестабильном кабеле).

## 🚀 3. Как прошить и запустить устройство

1. **Прошивка кода скетча:** Откройте файл `RehabDevice.ino` в Arduino IDE и нажмите кнопку **Загрузить (Upload)**.
2. **Прошивка файлов веб-интерфейса (`/data`):**
   * **Важно:** Закройте вкладку *Монитор порта (Serial Monitor)*, иначе Windows выдаст ошибку `PermissionError: Access is denied` (порт занят).
   * Нажмите кнопку **`Upload LittleFS to Pico/ESP8266/ESP32`** (или `ESP32 LittleFS Data Upload`). Утилита упакует папку `data` в образ и запишет во флэш-память.
3. **Первое подключение:**
   * Включите питание ESP32. При старте зажжётся точка доступа **`RehabDevice_AP`** (пароль: `rehab1234`).
   * Подключитесь к Wi-Fi с телефона или ноутбука — автоматически откроется страница авторизации (Captive Portal) или перейдите по адресу **`http://192.168.4.1`**.

## ⚙️ 4. Архитектура и принцип работы системы

Система состоит из четырёх взаимосвязанных программных модулей, работающих без блокировки основного цикла (`loop`):

```
+-------------------------------------------------------------------------+
|                              ESP32 CORE                                 |
|                                                                         |
|  +-----------------+       +-------------------+       +-------------+  |
|  |   SensorMPU     | ----> |  AnalyticsEngine  | ----> |  MemoryFS   |  |
|  | (DMP 6-Axis,    |       | (Амплитуда, тремор|       | (JSON файлы |  |
|  |  FIFO буфер)    |       |  сгибания, удерж.)|       |  сессий)    |  |
|  +-----------------+       +-------------------+       +-------------+  |
|          |                           |                        |         |
|          v                           v                        v         |
|  +-------------------------------------------------------------------+  |
|  |                       WebServerModule (Async)                     |  |
|  |           (WebSocket трансляция ~30 Гц + Captive Portal)          |  |
|  +-------------------------------------------------------------------+  |
+-------------------------------------------------------------------------+
                                       |
                                       v  (Wi-Fi 192.168.4.1)
                            [Смартфон / Планшет Врача]
```

### 🔹 1. Модуль опроса датчика (`SensorMPU.cpp`)
* Датчик MPU инициализируется с использованием встроенного сопроцессора **DMP (Digital Motion Processor)**, который аппаратно рассчитывает кватернионы ориентации с частотой 100 Гц.
* При готовности пакета датчик выдаёт импульс на пин `INT` (`GPIO 19`), после чего ESP32 мгновенно вычитывает FIFO-буфер без задержек и переводит кватернионы в углы Эйлера (Roll, Pitch, Yaw).
* Поддерживается **моментальная рекалибровка 0° на лету** (кнопка в интерфейсе принимает текущее положение руки за новый ноль).

### 🔹 2. Аналитический движок (`AnalyticsEngine.cpp`)
В реальном времени обрабатывает поток угловых данных и вычисляет медицинские метрики:
* **Амплитуда и границы:** Отслеживает минимальный и максимальный угол сгибания кисти за сессию.
* **Детекция тремора (Индекс плавности):** Вычисляет рывки углового ускорения (`d(omega)/dt`). При превышении порога рывка фиксируется микро-тремор, снижающий процент плавности.
* **Адаптивный детектор сгибаний:** Использует алгоритм гистерезиса для подсчёта реальных повторений упражнения, отсекая мелкое дрожание.
* **Время удержания:** Фиксирует секунды, проведённые пациентом в крайних точках сгибания (изометрическая нагрузка).

### 🔹 3. Хранилище сессий (`MemoryFS.cpp`)
* Каждая тренировка сохраняется как отдельный самодостаточный JSON-файл в директории `/sessions/` на флэш-памяти `LittleFS` (имя файла безопасно кодируется с меткой времени, а русское ФИО пациента `patientId` сохраняется внутри JSON).
* **Кольцевой буфер:** Перед каждой записью система проверяет свободную память. Если флэш-память заполнена более чем на 90%, самый старый файл тренировки автоматически удаляется, уступая место новым данным.

### 🔹 4. Веб-сервер и интерфейс (`WebServerModule.cpp` + `/data`)
* Раздаёт страницы по HTTP и поддерживает постоянное двухстороннее соединение по **WebSocket (`/ws`)**.
* **Интерфейс Пациента:** В браузере отрисовывается динамический круг с перекрестием, который синхронно вращается вслед за рукой пациента на скорости ~30 кадров в секунду.
* **Интерфейс Врача:** Выводит интерактивный столбчатый график (`Chart.js`), полную историю сохранённых сессий и позволяет **скачать всю базу данных в формате CSV** для отчётов и анализа в Excel.
